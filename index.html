What is the Problem?

The problem that I am approaching is a battle racing AI for my team project.
This AI will need to be able to find it's own way round teh track and interact with the players via steering the physics based car controller and using the weapons and powerups around the map

Why does the problem need solving?

The problem needs solving because the game is local multiplayer. If you are trying to play the game on your own then it would be very dull to play. There will need to be AI in the game to keep it interesting for the player. The AI will need to behave in a way that is similar to that of a real player to make it feel like a challenge

Which Technique to use:

The techniques i had considered using were GOAP and a more simple Utility based AI.
While learning about different AI techniques, these stood out to me because they did not seem too difficult to implement and they could provide the functionality that i was looking for. I could create an AI that would have a set of things that it could do and would react to the situation it was in and respond by picking actions to carry out.

Goap:

GOAP while still being a Utility based system, builds on it significantly
Using the GOAP system, each agent will have a set of actions that they can carry out and a set of goals that it tries to complete.
Here I have a diagram that shows 2 goals: Combat and Race. These goals consist of the actions that it takes to complete them.
The combat goal consists of the "Get Weeapon", "Find Nearest Player" and "Shoot Player" actions. The goal is complete when a player is shot.
The Race goal consists of the "Find Next Race Point" and "Drive To Next Point" actions. The goal is complete when the car has driven to the point.
The agent would have a set of biases for these goals as to which goal it tries to complete. Each action will also have a cost to complete, this allows the agent to work out what it should do. If the agent had no biases then it would do the same thing every time when given the same circumstances. The biases allow for variance.

Utility

Utility based AI is a system that uses cost analysis to pick the most efficient actions to take.
The diagram to the right shows a scenario where an agent is running out of food and needs more. They can either go to the shops and buy the food or they can order the food online.

Going to get food:
If the agent goes to get food themself then the costs they will encounter are: £ 1.50 fuel there and back and £50 for food.
The total cost for this is: (£1.5 x 2) + 50 = 53.
Ordering online:
If the agent decides to order online then the costs are: £50 for food and the £2 delivery fee.
The total cost for ordering online is: 50 + 2 = 52.

The more efficient action in this instance is to order the food online since the petrol costs outweight the delivery costs. However, if the agent is in dire need of food then they might not be able to wait for the food. if we add the 72 hour delivery time to the cost for ordering online, the cost is increased to 52 + 72 = 124. This would make going to get the food the more efficient action.
Biases
Biases can be added to the agents calculations. If we add a bias to the agent to have a strong preferance against travel then we can change the calculation. If we cost travel at £30/hour and the trip would take 1.5 hours each way then the calculation becomes:
(£1.5 x 2) + 50 + (1.5hr x 2 x 30) = 143.
After adding the bias, the more efficient action becomes ordering online again.
