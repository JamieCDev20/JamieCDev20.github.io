I started an Inverse Kinematics (IK) project over the summer in my free time.
I was just experimenting to see what I could come up with using an algorithm called
Cyclic Coordinate Descent (CCD).

CCD loops through the IK chains (limbs) from tip to root and rotates each joint to move the tip
closer to the target.

I got this working with Unity's transforms and eventually using a 3D spider model made by
Matthew Brett. A friend and colleague




The above and below gifs show the spider walking around a small demo scene.
As well as my first attempt at getting the legs to respect terrain like
slopes and steps.




After dropping this project for a while, I restarted it with more research going
into the project at first. I settled on an algorithm called
Forward And Backward Reaching Inverse Kinematics (FABRIK).

Using the papers and videos made available by the creator, found here, I was
able to form my own implementation of the algorithm.

FABRIK works by looping through the limb twice, once from tip to root:
rotating each joint to point towards it's target, moving it to it's target
and then setting it'self as the next joint's target.
Then a second time from root to tip:
Translating each joint back towards the root so that the limb is where it started.

Using a 3D Orc model made by another friend and colleague Louise Crouch.
I was able to create a walk cycle where all of the motion is generated
by mathematics.
